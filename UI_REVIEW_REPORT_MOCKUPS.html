2. SVG Icons Inline PATTERN

Centinaia di righe di SVG inline copiate nei componenti. Aumenta la dimensione del bundle, rende il codice illeggibile e duplica definizioni identiche su pagine diverse.
Before - 130+ Righe SVG Inline
{'/* AuctionRoom.tsx - righe 45-175 */'} const icons = ( <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="2"> <path d="M12 2L2 7l10 5 10-5-10-5z"/> <path d="M2 17l10 5 10-5"/> <path d="M2 12l10 5 10-5"/> </svg> <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none"> <circle cx="12" cy="12" r="10"/> <path d="M12 6v6l4 2"/> </svg> {/* ... altre 110 righe ... */} <svg viewBox="0 0 24 24"> <path d="M19 21l-7-5-7 5V5a2 2..."/> </svg> ); {/* Duplicato in 6 file diversi */}
~130 righe di SVG, ripetute in 6 componenti
After - Lucide React Icons
import { Timer, Users, Trophy, Gavel, TrendingUp, Wallet, Shield, Clock } from 'lucide-react'; // Utilizzo pulito nel JSX <Timer size={20} className="text-blue-400" /> <Trophy size={20} className="text-amber-400" /> <Gavel size={20} className="text-green-400" /> // Tree-shaking automatico // Solo le icone usate nel bundle // ~0.5KB per icona vs ~2KB inline
8 righe di import, 200+ icone disponibili, tree-shaking
