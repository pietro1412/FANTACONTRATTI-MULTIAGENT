Panoramica

⚽ Cos'e il Primo Mercato

Il Primo Mercato Assoluto e la prima asta della lega. Tutti i manager acquistano giocatori per riempire gli slot della propria rosa. Al termine, non si ripresentera mai piu per quella lega.

Precondizioni

  • ✓ La lega e stata creata e avviata dall'admin
  • ✓ Numero manager pari, compreso tra 6 e 20
  • ✓ Slot ruoli definiti (default: 3P, 8D, 8C, 6A = 25 giocatori)
  • ✓ Budget iniziale configurato (default: 500)

Meccanica dell'Asta

Ordine di Chiamata

  • ↻ L'admin definisce l'ordine dei turni (o sceglie random)
  • ⚽ Ogni manager, a turno, sceglie un giocatore svincolato da mettere all'asta
  • ↺ Il turno ruota ciclicamente tra tutti i manager attivi

Vincolo Slot per Ruolo

🔒 Regola Fondamentale

L'asta procede per ruolo in ordine: P → D → C → A

Finche TUTTI i manager non hanno completato gli slot di un ruolo, non si puo passare al ruolo successivo.

📊 Esempio: Slot Portieri (3 slot)
ManagerPortieriStato
Manager A3/3✓ Completato
Manager B2/3✕ Manca 1
Manager C3/3✓ Completato
→
Si resta sui Portieri finche Manager B arriva a 3/3. Poi si passa automaticamente ai Difensori.

Chi Puo Fare Offerte

  • ✓ Tutti i manager possono fare offerte per qualsiasi giocatore all'asta
  • ✕ Slot pieno: se un manager ha gia riempito gli slot del ruolo corrente, NON puo offrire
  • ✕ Budget insufficiente: se il bilancio < offerta + ingaggio default, viene escluso

Meccanica Offerta

💰 Base d'Asta
1

Ogni asta parte da un'offerta minima di 1 credito.

⏱ Timer
  • ⏱ Configurabile dall'admin
  • ↺ Reset ad ogni nuova offerta
  • 🏆 Vincitore = ultimo offerente allo scadere
💡
Conferma chiamata: quando un manager sceglie il giocatore da astare, il sistema chiede conferma per evitare errori di click.

Acquisto e Contratto

Contratto Default

Alla vittoria dell'asta, il giocatore riceve automaticamente un contratto default:

Ingaggio Math.max(1, Math.round(prezzo / 10))
Durata Sempre 3 semestri
🔗
Vedi Bibbia CONTRATTI sezione 3 per la tabella fasce completa (1-4→1, 5-14→1, 15-24→2, ...).

Verifica Budget Pre-Offerta

Requisito per fare offerta bilancio ≥ offerta + ingaggio_default
⚽ Esempio: Esposito
1
Manager vuole offrire 75
Ingaggio default = Math.max(1, Math.round(75/10)) = 8
2
Bilancio minimo richiesto
75 + 8 = 83
3
Se il manager ha bilancio ≥ 83, puo offrire
Se bilancio < 83, l'offerta viene bloccata

Modifica Post-Acquisto

Il vincitore puo modificare il contratto immediatamente dopo l'acquisto:

  • ↑ Solo in aumento (ingaggio, oppure ingaggio + durata). La durata da sola NON puo essere aumentata.
  • 💰 Se aumenta l'ingaggio, il costo effettivo = prezzo_asta + nuovo_ingaggio
  • 💡 La durata non impatta il costo immediato
🔗
Vedi Bibbia CONTRATTI sezione 4 per le regole di modifica complete.

Impatto Finanziario

Per Ogni Acquisto

Effetto sull'acquirente Budget -= prezzo_asta
Effetto sul monte ingaggi Monte Ingaggi += ingaggio_effettivo (default o modificato)
Bilancio risultante Bilancio = Budget − Monte Ingaggi

Poteri dell'Admin

⏱ Gestione Timer

L'admin puo modificare la durata del timer dell'offerta durante l'asta.

L'admin puo bloccare il timer in pausa in qualsiasi momento.

⏸ Pausa

Un manager puo chiedere una pausa (urgenza):

  • ⏸ Pausa chiamata: ferma la scelta del giocatore
  • ⏸ Pausa asta: ferma l'asta in corso

Rettifiche

⚠ L'admin deve sempre poter correggere
Azione AdminEffetto
Annulla fine asta Riprende dall'ultima offerta (timer riparte)
Annulla chiamata Il giocatore torna svincolato, il turno resta al manager che deve scegliere
Rettifica transazione Asta conclusa annullata: giocatore torna libero, crediti restituiti al manager

Fine Asta

✓
L'admin dichiara conclusa l'asta quando tutti i manager hanno riempito tutti gli slot.

Manager Escluso

⚠ Budget Insufficiente

Se un manager non ha bilancio sufficiente per fare offerte:

Soglia minima bilancio < offerta minima (1) + ingaggio minimo (1) = 2
  • ✕ Viene escluso dall'asta
  • ✕ Non partecipa ai turni di chiamata
  • ? I suoi slot rimanenti restano vuoti (da definire con admin)
