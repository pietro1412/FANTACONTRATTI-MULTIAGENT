Fonti Dati

📄 Fantagazzetta (Quotazioni)

File Excel caricato dal SuperAdmin con i dati anagrafici e le quotazioni dei giocatori.

Modello DB SerieAPlayer

Dati: Nome, squadra, ruolo, quotazione, ID interno

Frequenza: A ogni sessione di mercato

⚽ API Football (Statistiche)

Provider api-sports.io (v3) per statistiche in tempo reale dei giocatori.

Link nel DB SerieAPlayer.apiFootballId

Dati: Presenze, gol, assist, voto medio, cartellini, etc.

Lega: Serie A (ID 135), Stagione 2025

Algoritmo di Matching

Il matching associa i giocatori del DB (Fantagazzetta) ai giocatori di API Football tramite il nome. Non esiste un ID comune tra le due fonti.

Livelli di Matching

Livello Strategia Confidenza Auto-match
1 Match esatto nome completo HIGH Si (se unico)
2 Match esatto cognome HIGH Si (se unico)
3 Match su parte del nome HIGH Si (se unico)
4 Match parziale inclusione cognome MEDIUM Si (se unico)
5 Levenshtein distance ≤ 20% LOW Si (se unico)
6 Match multipli (ambiguo) — No → SuperAdmin
7 Nessun match NONE No → SuperAdmin

Strategy A: Match Esatto (Livelli 1-2)

💡 Esempio
Confronto DB: "Morata" → normalize → "morata"
API: "Morata" → normalize → "morata"
→ Match esatto cognome: AUTO

Confronta nome completo normalizzato E cognome estratto. Se esattamente 1 match nella squadra → auto-match.

Strategy B: Match Parziale Cognome (Livello 4)

💡 Esempio
Inclusione DB: "Di Lorenzo" → lastName → "lorenzo"
API: "Giovanni Di Lorenzo" → lastName → "lorenzo"
→ "lorenzo".includes("lorenzo"): MATCH

Verifica se un cognome contiene l'altro (o viceversa). Utile per abbreviazioni o nomi composti.

Strategy C: Name-Part Match (Livello 3)

💡 Esempio
Parti del nome DB: "Morata" → parts → ["morata"]
API: "Alvaro Morata" → parts → ["alvaro", "morata"]
→ "morata" in apiParts: MATCH

Divide entrambi i nomi in parti (≥ 3 caratteri). Se una parte del DB compare nelle parti API e c'e un solo match → auto-match.

Strategy D: Levenshtein (Livello 5)

💡 Esempio
Distanza di editing DB: "Ronalod" → lastName → "ronalod"
API: "Ronaldo" → lastName → "ronaldo"
→ distance = 2, maxLen = 7, ratio = 28% > 20%: NO MATCH

Soglia conservativa al 20% (i proposals usano il 30%). Solo se esattamente 1 match → auto-match.

Normalizzazione Nomi

La funzione normalizeName() esegue:

1
Lowercase
"Álvaro" → "álvaro"
2
Rimozione diacritici
"álvaro" → "alvaro" (NFD + strip combining marks)
3
Solo lettere e spazi
"o'brien" → "obrien"
4
Trim spazi
" alvaro " → "alvaro"

Persistenza Match

💾 Match Automatici

Salvati immediatamente in SerieAPlayer.apiFootballId.

Ai run successivi, i giocatori gia matchati vengono skippati. Se un API Football ID e gia assegnato ad un altro giocatore, viene skippato (unique constraint).

👤 Match Manuali

Il SuperAdmin seleziona il match corretto dalla UI dei proposals.

Salvato tramite manualMatch(). Permanente: non viene sovrascritto da run automatici successivi.

🔎 Match Proposals (UI Assistita)

Per giocatori ambigui o senza match, il sistema genera proposals con confidenza HIGH/MEDIUM/LOW/NONE.

Il SuperAdmin conferma o cerca manualmente il giocatore API tramite la funzione di ricerca.

Flusso Operativo SuperAdmin

Pipeline Completa

STEP 1 CACHE — Refresh cache squadre API Football (scarica rose aggiornate)
↓
STEP 2 MATCH — Matching automatico (Strategy A → B → C → D)
↓
STEP 3 PROPOSALS — Gestione ambigui e non trovati (UI SuperAdmin)
↓
STEP 4 SYNC STATS — Sincronizza statistiche stagione corrente

Frequenza Consigliata

Operazione Quando
Cache refreshA inizio stagione + dopo mercato invernale
Match automaticoDopo ogni cache refresh
Match manualeDopo ogni match automatico (per risolvere ambigui)
Sync statisticheSettimanalmente durante la stagione

Limiti API

⚠
Piano Free: 100 chiamate/giorno
Cache refresh: ~20 call | Match: 0 call (usa cache) | Sync stats: ~30 call
Totale pipeline completa: ~50 call

Gestione Errori

🔑 API Key Non Configurata

Messaggio: "API_FOOTBALL_KEY non configurata"

Soluzione: Impostare la variabile d'ambiente API_FOOTBALL_KEY nel server.

⏳ Rate Limiting

Delay di 100ms tra le chiamate alle squadre. Se il piano API viene superato, la chiamata fallisce con errore HTTP.

❓ Giocatori Non Matchabili

Giocatori con nomi molto diversi tra le due fonti (soprannomi), o nuovi non ancora nelle rose API Football.

Soluzione: Match manuale via UI SuperAdmin con funzione di ricerca.
