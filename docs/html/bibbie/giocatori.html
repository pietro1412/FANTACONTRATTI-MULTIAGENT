Transizioni di Stato

IN_LISTA→ ESTERO (Super Admin, post aggiornamento quotazioni)
IN_LISTA→ RETROCESSO (Super Admin, fine stagione)
IN_LISTA→ RITIRATO (Super Admin)
IN_LISTA→ SVINCOLATO (Sistema: scadenza contratto o taglio)
SVINCOLATO→ IN_LISTA (Sistema: acquistato da un manager)

Chi Gestisce gli Stati Globali

👑
Il Super Admin (non l'admin di lega) imposta ESTERO, RETROCESSO, RITIRATO. Questi stati valgono per tutte le leghe.

L'aggiornamento avviene in 2 step:

STEP 1Import nuovo file quotazioni Fantagazzetta → sistema rileva giocatori ceduti/assenti
↓
STEP 2Il Super Admin conferma e imposta lo stato specifico (ESTERO vs RETROCESSO vs RITIRATO)
4

Aggiornamento Dati

📄 File Quotazioni Fantagazzetta
  • 👤Import manuale da parte del Super Admin
  • 🔄Aggiorna la lista giocatori (nuovi arrivi, cessioni)
  • ⚠I dati possono cambiare durante un mercato ricorrente aperto
⚽ API Football
  • 📈Chiamate API per statistiche aggiornate
  • 🔄Aggiornamento squadra (calciomercato)
  • 📷Foto e dati anagrafici

Calciomercato Reale

Quando un giocatore cambia squadra nella realta:

1Viene aggiornato il file quotazioni Fantagazzetta
↓
2Vengono chiamate API Football aggiornate
↓
3Il sistema riflette il nuovo stato/squadra

Impatto sulle Leghe

✈ Giocatore ESTERO

In ogni lega dove e sotto contratto:

  • ✕Deve essere rilasciato (draftReleased: true + draftExitDecision: 'RELEASE')
  • ✓Costo taglio: 0
  • 💰Indennizzo: valore impostato dall'admin (min 0)
📉 Giocatore RETROCESSO

In ogni lega dove e sotto contratto:

  • ✕Deve essere rilasciato
  • ✓Costo taglio: 0
  • ✕Indennizzo: nessuno
👤 Giocatore SVINCOLATO (nella Lega)
  • ✓Disponibile per acquisto nelle aste svincolati
  • ✕Non ha contratto attivo con nessun manager della lega
💡
Per il release di giocatori ESTERO/RETROCESSO servono ENTRAMBI i flag: draftReleased: true e draftExitDecision: 'RELEASE'.
